# Scenario: User Receives “Access Denied” in SharePoint

## Issue
User reports receiving an “Access Denied” message when attempting to access a SharePoint site.

## Possible Causes
- User not added to site permissions
- Incorrect permission level assigned
- Permissions inheritance broken
- Access via incorrect URL

## Troubleshooting Steps
1. Confirm user identity and email
2. Check SharePoint site permissions
3. Verify group membership
4. Check if inheritance is broken
5. Confirm access via correct site URL

## Resolution
Add user to the appropriate SharePoint group and confirm access.
